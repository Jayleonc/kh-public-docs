Knowledge Hub 面向企业知识场景，目标是把文档接入、知识组织、检索、权限和 AI 回答放进同一条受控链路。这个站点使用公开文档构建：Markdown 文档通过 Git Sync 进入知识空间，再由 Agent 在公开资料范围内完成问答。

## 三句话先认识这个项目

- Knowledge Hub 是一套自研企业知识智能平台，核心任务是把文档变成可检索、可引用、守权限的知识系统。
- 它的问答主路径已经演进到 Agent-first 形态，检索、全文读取、目录浏览和澄清都是受控工具。
- 这个公开站点使用 Knowledge Hub 自身承载文档阅读和问答能力。

## 这份站点包含什么

当前公开文档主要覆盖下面这些内容：

- 文档如何成为唯一可信源
- 权限如何进入检索链路，而不是只停留在页面层
- Agent 如何在受控工具集内完成多步问答
- 回答如何回到原始文档，形成可核验的证据链
- 这个项目如何借助 AI 开发和持续演进
- 它当前已经承担哪些实际使用场景
- 公开访问环境如何与内部知识环境隔离

## 公开范围与回答边界

当前公开站点只允许基于这套公开文档回答问题。没有写进公开文档的客户信息、内部配置、部署凭据和未发布能力，都不应进入回答。

企业知识系统常见的问题之一，是模型可用信息与系统允许暴露的信息不一致。Knowledge Hub 会在公开环境里主动把这个边界收紧。

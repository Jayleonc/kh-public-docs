企业知识系统常见的问题不是缺少文档，而是文档无法被稳定使用。

文档可能分散在多个仓库、网盘、系统和聊天记录里；用户不知道该看哪一份；AI 即使能回答，也可能没有引用、没有权限边界、没有更新机制。Knowledge Hub 的目标是把这些问题收束到一条可管理的链路里。

## 文档分散

文档可能来自 Markdown、PDF、接口契约、产品说明、运营材料和历史方案。Knowledge Hub 通过项目和虚拟文件树组织文档，让用户能按路径阅读，也能通过问题检索。

## 更新不可控

手动上传适合临时材料，但不适合作为正式客户文档的长期维护方式。正式文档应该通过 Git 审查、合并和同步，保证每次变更都有记录，也可以快速回滚。

## 回答缺少来源

企业场景不能只看 AI 回答是否自然，还要看它是否基于授权文档。Knowledge Hub 的回答会携带引用信息，让用户可以回到原始文档确认上下文。

## 权限容易遗漏

很多知识库系统只在页面层做权限控制，但检索结果可能绕过页面权限。Knowledge Hub 将访问控制下沉到文档、节点和检索链路中，减少越权检索风险。

## AI 决策链路不透明

固定 RAG 流程适合简单问答，但复杂问题往往需要多步检索、澄清、全文读取和来源比对。Knowledge Hub 使用 Agent 工具化方式，让模型可以在受控工具范围内完成多轮查询。

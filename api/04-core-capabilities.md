description: "Knowledge Hub 面向客户的主要能力清单。"

---

Knowledge Hub 的能力可以分为五层：内容接入、知识组织、检索问答、权限治理和系统集成。

## 内容接入

- Markdown 文档同步
- PDF 等文档解析
- 图片和多模态内容处理
- Git 仓库同步
- 手动上传作为补充入口

## 知识组织

- 项目级知识空间
- 虚拟文件树
- 文档路径和引用定位
- 文档状态与索引状态分离
- 文档摘要和结构化切片

## 检索问答

- 向量检索
- 关键词检索
- 混合检索
- 上下文扩展
- 多轮对话
- 来源引用
- 文档全文读取

## 权限治理

- 多租户隔离
- RBAC 权限控制
- 文档和目录级访问策略
- 检索阶段权限过滤
- API Key 和 JWT 两类访问方式

## 系统集成

- Web Chat 对外文档查询
- Web Workspace 内部知识工作台
- API 接入业务系统
- Git Sync 文档发布链路
- 审计和调试链路

## 对外推荐形态

对外客户访问建议使用 Web Chat。它只暴露阅读和问答能力，不暴露上传、空间设置和权限管理。

文档维护建议使用 Git Sync。它能让客户可见内容经过代码审查和版本管理，再同步到 Knowledge Hub。

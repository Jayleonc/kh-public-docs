Knowledge Hub 采用分层架构。前端提供面向不同角色的入口，后端负责认证、权限、文档、检索、Agent 和同步任务，底层使用 PostgreSQL、pgvector、Redis 和文档解析服务支撑。

## 前端入口

**web-chat**

面向外部客户和最终用户，提供文档阅读、全文搜索、对话问答和引用查看。

**web-workspace**

面向内部运营和业务团队，提供空间创建、文档上传、知识问答和协作管理。

**web-admin**

面向技术管理员，负责租户、项目、权限、配置、审计和系统管理。

## 后端核心模块

**Project**

以项目为核心组织知识空间，维护虚拟文件树、目录、文件节点和访问策略。

**Document**

负责文档接入、解析状态、索引状态、内容存储和文档元信息。

**RAG**

负责向量化、检索、重排、上下文扩展和召回结果组装。

**Agent**

负责多轮推理和工具调用。RAG 是 Agent 可调用的工具之一，而不是唯一入口。

**ACL / RBAC**

负责权限决策和访问控制。权限不仅影响页面展示，也影响检索范围。

**Git Sync**

负责从 Git 仓库同步文档、创建 VFS 节点、更新文档内容，并触发后续索引。

## 数据与基础设施

- PostgreSQL 存储业务数据
- pgvector 存储向量索引
- Redis 支撑缓存和异步任务
- 文档解析服务处理 PDF、图片和复杂格式
- LLM Provider 提供生成和推理能力

## 设计原则

Knowledge Hub 的核心设计原则是：项目优先、权限优先、来源可追溯、文档可同步、回答可验证。

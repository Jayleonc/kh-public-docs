Knowledge Hub 的安全设计围绕一个原则：用户只能看到和检索到自己被授权访问的内容。

## 多租户隔离

每个租户拥有独立的项目、文档、配置和访问策略。请求进入系统后，会先经过认证和租户识别，再进入后续业务逻辑。

租户隔离不仅用于页面展示，也用于数据库查询、项目访问、文档访问和检索范围控制。

## RBAC

RBAC 用于控制用户能执行什么操作，例如：

- 查看项目
- 创建空间
- 上传文档
- 修改权限
- 管理租户配置
- 查看审计日志

对外客户入口一般不需要管理权限，只需要只读和问答权限。

## ACL

ACL 用于控制具体资源能被谁访问。资源可以是项目、目录或文档。访问策略可以表达公开、私有、指定成员、指定部门或指定用户组等场景。

## 检索阶段权限过滤

知识问答不是简单读取页面上可见的内容。系统在检索阶段也会执行权限过滤，避免用户通过提问绕过文档权限。

这意味着没有权限的文档不应该出现在召回结果里，也不应该成为 AI 回答依据。

## 对外 Demo 的隔离建议

公开文档站点建议使用独立租户和独立项目：

- 租户只放公开材料
- 项目只绑定公开文档仓库
- API Key 只授予只读和问答权限
- 禁止外部用户上传或修改文档
- 开启限流和审计

这样即使外部链接被转发，也只会访问公开资料。

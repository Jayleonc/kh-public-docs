## Knowledge Hub 是知识库还是 Chatbot？

它不是单纯的 Chatbot。Knowledge Hub 同时包含文档管理、虚拟文件树、索引、检索、权限、审计和 Agent 问答。Chat 只是其中一个用户入口。

## 为什么对外入口推荐 Web Chat？

外部用户主要需要阅读和提问，不需要上传文档、配置权限或管理空间。Web Chat 的边界更清晰，也更适合公开访问。

## Web Workspace 为什么不直接给客户用？

Web Workspace 是内部知识工作台，包含空间创建、上传、成员管理和信源选择等能力。公开场景下暴露这些能力会增加权限和操作复杂度。

## 为什么正式文档推荐 Git Sync？

正式文档需要审查、版本记录、回滚和发布流程。Git Sync 可以让 Git 仓库成为唯一可信源，避免手动上传带来的内容漂移。

## 文档更新后多久可以问到新内容？

这取决于同步和索引任务完成时间。一般流程是先同步 Git 文档，再完成解析和索引。索引完成后，新内容才能稳定参与问答。

## AI 回答会不会使用公开文档之外的信息？

对外 Demo 应该限制在公开文档范围内。没有写进公开文档的内部信息，不应该成为回答依据。

## 如果文档里没有答案会怎样？

理想回答应该明确说明当前公开文档中没有找到相关信息，而不是自行编造。

## 是否支持权限隔离？

支持。Knowledge Hub 通过租户、项目、RBAC 和 ACL 控制访问范围，并在检索阶段执行权限过滤。

## 是否只能使用 Markdown？

不是。Knowledge Hub 可以接入多种文档类型。对于对外文档站点，Markdown 更适合作为主格式，因为它容易审查、版本管理和 Git Sync。
